Persistencia & JPA

Jakarta Persistence API (JPA) es la especificación estándar para la gestión de persistencia y mapeo objeto-relacional en Java. Permite almacenar y recuperar objetos Java en bases de datos relacionales sin escribir SQL explícito, mediante anotaciones declarativas.

Ventajas clave
  • Abstracción del acceso a datos con anotaciones estándar.
  • Gestión automática del ciclo de vida de las entidades.
  • Portabilidad entre proveedores: Hibernate, EclipseLink, OpenJPA.
  • Consultas orientadas a objetos mediante JPQL y Criteria API.

Entidades JPA

Una entidad es una clase POJO anotada con @Entity que se mapea a una tabla de base de datos. Debe tener un constructor sin argumentos y una clave primaria con @Id.

import jakarta.persistence.*;

@Entity
@Table(name = "empleados")
public class Empleado {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 100)
    private String nombre;

    @Column(unique = true)
    private String email;

    @Basic(fetch = FetchType.LAZY)
    private String foto;

    @Transient
    private String campoNoPersistente;
    
    // constructores, getters y setters
}
@Entity @Table @Id @GeneratedValue @Column @Lob

Unidad de persistencia (persistence.xml)

El archivo META-INF/persistence.xml define la unidad de persistencia: entidades gestionadas, proveedor JPA, tipo de transacción y propiedades de conexión JDBC.

<persistence version="3.0" xmlns="https://jakarta.ee/xml/ns/persistence">
    <persistence-unit name="miUnidadPU" transaction-type="RESOURCE_LOCAL">
        <provider>org.hibernate.jpa.HibernatePersistenceProvider</provider>
        <class>com.example.Empleado</class>
        <properties>
            <property name="jakarta.persistence.jdbc.url" value="jdbc:postgresql://localhost:5432/empresa"/>
            <property name="jakarta.persistence.jdbc.user" value="appuser"/>
            <property name="jakarta.persistence.schema-generation.database.action" value="update"/>
        </properties>
    </persistence-unit>
</persistence>

EntityManager y transacciones

EntityManager es el núcleo de JPA: gestiona el contexto de persistencia, ciclos de vida y operaciones CRUD. En Java SE usamos EntityTransaction para controlar transacciones.

EntityManagerFactory emf = Persistence.createEntityManagerFactory("miUnidadPU");
EntityManager em = emf.createEntityManager();
EntityTransaction tx = em.getTransaction();

try {
    tx.begin();
    Empleado emp = new Empleado();
    emp.setNombre("Ana Torres");
    em.persist(emp);
    tx.commit();
} catch (Exception e) {
    if (tx.isActive()) tx.rollback();
} finally {
    em.close();
    emf.close();
}
Métodos principales:
  • persist() – insertar entidad
  • find(Class, pk) – buscar por clave primaria
  • merge() – sincronizar entidad detached
  • remove() – eliminar entidad gestionada

Operaciones CRUD

JPA simplifica el acceso a datos con operaciones directas sobre entidades. El dirty checking actualiza automáticamente los cambios dentro de una transacción activa.

OperaciónMétodo JPADescripción

Readem.find(Class, id)Recupera entidad por clave primaria
Updateem.merge(entity) / dirty checkingActualiza entidad desacoplada o modifica managed

JPQL: Java Persistence Query Language

Lenguaje de consultas orientado a objetos, trabaja sobre entidades y sus atributos. Soporta joins, funciones agregadas, parámetros nombrados y actualizaciones masivas.

// Consulta tipada con parámetro
TypedQuery<Empleado> query = em.createQuery(
    "SELECT e FROM Empleado e WHERE e.departamento.nombre = :dept", Empleado.class);
query.setParameter("dept", "TI");
List<Empleado> resultados = query.getResultList();

// Actualización masiva
int updated = em.createQuery("UPDATE Empleado e SET e.sueldo = e.sueldo * 1.05 WHERE e.antiguedad > 3")
                .executeUpdate();

// Proyección con constructor
List<EmpleadoDTO> dtos = em.createQuery(
    "SELECT NEW com.example.EmpleadoDTO(e.nombre, e.email) FROM Empleado e", EmpleadoDTO.class)
    .getResultList();

Herencia en JPA

JPA soporta jerarquías de herencia con las estrategias definidas en @Inheritance. La elección afecta al rendimiento y diseño de tablas.

SINGLE_TABLE

Una sola tabla para toda la jerarquía. Usa columna discriminadora. Rendimiento óptimo en consultas polimórficas.

JOINED

Tablas normalizadas (una por clase). Joins en consultas, evita columnas nulas. Ideal para jerarquías complejas.

TABLE_PER_CLASS

Una tabla por clase concreta. Consultas polimórficas poco eficientes (UNION). Menos recomendada.

Asociaciones y cardinalidad

Las relaciones entre entidades se definen con anotaciones que expresan la navegabilidad y cardinalidad: @OneToOne, @OneToMany, @ManyToOne, @ManyToMany.

@Entity
public class Departamento {
    @Id private Long id;
    @OneToMany(mappedBy = "departamento", cascade = CascadeType.ALL)
    private List<Empleado> empleados;
}

@Entity
public class Empleado {
    @Id private Long id;
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "depto_id")
    private Departamento departamento;
}
  • @ManyToOne : FK en la tabla de la entidad origen
  • @OneToMany : colección en el lado inverso
  • @JoinColumn : personaliza nombre de columna FK
  • @JoinTable : para relaciones ManyToMany
@ManyToOne @OneToMany @ManyToMany

Generación desde BD y herramientas JPA

Los IDEs modernos permiten ingeniería inversa: generar entidades JPA directamente desde un esquema relacional, incluyendo relaciones, persistence.xml y controladores JPA (facades).

NetBeans / Eclipse Dali

Asistentes "Entidades desde base de datos". Generación automática de clases @Entity, controladores y mapeo de relaciones.

IntelliJ IDEA Ultimate

JPA Buddy / Persistence → Generate Entities from Schema. Soporte visual para JPQL, persistence.xml y anotaciones.

Hibernate Tools (Maven)

Reverse engineering con archivo reveng.xml. Generación de entidades anotadas y mapeos HBM.

Controladores JPA: Clases que encapsulan el EntityManager y ofrecen métodos CRUD genéricos. Los IDEs pueden generarlos automáticamente (patrón DAO/Facade).
