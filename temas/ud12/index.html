Collections
Framework

Estructuras de datos estándar en Java: listas, conjuntos, mapas, iteradores y genéricos.

Javadoc oficial · java.util (Java 21)
Descripción: Colecciones ordenadas que permiten elementos duplicados y acceso por índice.
Casos de uso: Mantener secuencias, listas de tareas, historial de acciones.
ArrayList ↗ Acceso O(1) por índice, inserción al final eficiente. Uso general.
LinkedList ↗ Inserciones/eliminaciones en medio O(1). También implementa Deque.
Vector ↗ Como ArrayList pero sincronizado. Legado, preferir ArrayList.
Stack ↗ LIFO — algoritmos de retroceso, pilas de ejecución.

Descripción: Colecciones sin duplicados. Sin orden garantizado salvo SortedSet.
Casos de uso: Eliminar duplicados, verificar pertenencia, operaciones matemáticas de conjuntos.
HashSet ↗ O(1) para add/contains/remove. Sin orden de elementos.
LinkedHashSet ↗ Mantiene orden de inserción. Ligeramente más lento que HashSet.
TreeSet ↗ Orden natural o con Comparator. Operaciones en O(log n).
EnumSet ↗ Implementación muy eficiente para tipos enum. No acepta null.

Descripción: Pares clave → valor para acceso rápido mediante clave única.
Casos de uso: Contadores, diccionarios, configuración, caché en memoria.
HashMap ↗ O(1) amortizado. Sin orden de claves. El más usado.
LinkedHashMap ↗ Mantiene orden de inserción. Ideal para caches LRU.
TreeMap ↗ Claves ordenadas O(log n). Soporta subMap/headMap/tailMap.
ConcurrentHashMap ↗ Thread-safe moderno. Preferir sobre Hashtable.

// Método genérico acotado — T debe ser Comparable
public static <T extends Comparable<T>> T max(List<T> lista) {
  return lista.stream()
               .max(Comparator.naturalOrder())
               .orElseThrow();
}
