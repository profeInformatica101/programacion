Introducción
¿Qué es un fichero binario?
Fundamentos java.io

Todo fichero es una secuencia de bytes. Los ficheros binarios codifican los datos en formatos específicos — píxeles de imagen, muestras de audio, objetos Java — que no son texto legible directamente.

Objeto Java
→

0x4A 0xFE…
→

Jerarquía de streams · InputStream / OutputStream son la raíz de toda la E/S binaria en Java.

EJEMPLOS DE FICHEROS BINARIOS

PNG  Compresión sin pérdida · Canal alpha · Cabecera 8 bytes + chunks IHDR/IDAT

BMP  Sin compresión · Píxeles en bruto · Cabecera fija de 54 bytes (ideal para RandomAccessFile)
.dat / .bin  Objetos Java serializados — formato binario propietario de la JVM
02
ImageIO
Leer y escribir imágenes
javax.imageio PNG · JPEG · BMP
Javadoc

ImageIO soporta PNG, JPEG y BMP sin dependencias externas. Devuelve un BufferedImage con acceso a nivel de píxel.

import javax.imageio.ImageIO;
import java.awt.image.BufferedImage;
import java.io.File;

BufferedImage img = ImageIO.read(new File("imagen.png"));
System.out.println(img.getWidth() + "x" + img.getHeight());

// Leer color de un píxel (int ARGB)
int rgb  = img.getRGB(10, 20);
int rojo = (rgb >> 16) & 0xFF;

// Guardar en otro formato
ImageIO.write(img, "jpg", new File("salida.jpg"));

03
FileInputStream
Lectura secuencial de bytes
java.io try-with-resources

Lee el fichero byte a byte (o en bloques). Siempre leer en bloques y usar try-with-resources.

try (FileInputStream fis = new FileInputStream("imagen.bmp")) {
    byte[] buffer = new byte[4096];
    int leidos;
    while ((leidos = fis.read(buffer)) != -1) {
        for (int i = 0; i < leidos; i++) {
            System.out.printf("%02X ", buffer[i] & 0xFF);
        }
    }
} // fis.close() automático

Serialización
Persistir objetos completos
ObjectOutputStream Serializable
Javadoc 
public class Empresa implements Serializable {
    private static final long serialVersionUID = 1L;
    private String nombre;
    private transient String passTemp; // excluida
}
// Guardar
try (ObjectOutputStream out = new ObjectOutputStream(
        new FileOutputStream("empresa.dat"))) {
    out.writeObject(miEmpresa);
}
// Recuperar
try (ObjectInputStream in = new ObjectInputStream(
        new FileInputStream("empresa.dat"))) {
    Empresa e = (Empresa) in.readObject();
}

CONCEPTOS CLAVE

Serializable
Interfaz marcador

No tiene métodos. Su presencia autoriza a la JVM a serializar el objeto. Sin ella se lanza NotSerializableException.

serialVersionUID
Control de versiones

Identifica la versión de la clase. Si cambia sin actualizar el UID, la deserialización lanzará InvalidClassException.

transient
Campos excluidos

Excluye el campo del proceso. Útil para contraseñas, conexiones a BD o datos que no deben persistir.

05
RandomAccessFile
Acceso directo a cualquier byte
seek() Little-endian 
try (RandomAccessFile raf =
        new RandomAccessFile("imagen.bmp", "rw")) {
    raf.seek(18); // offset 18 = ancho (little-endian)
    int ancho = Integer.reverseBytes(raf.readInt());
    raf.seek(22); // offset 22 = alto (little-endian)
    raf.writeInt(Integer.reverseBytes(50));
    System.out.println("Ancho: " + ancho + ", bytes: " + raf.length());
}

CASOS DE USO DE RandomAccessFile

Editar cabecera BMP

Modifica ancho, alto o bits por píxel saltando directamente al offset sin leer todos los datos.

Registros de tamaño fijo

El registro n está en offset n × tamRegistro. Acceso O(1) con seek().

Tags ID3 en MP3

Metadatos ID3 en los últimos 128 bytes del MP3. Con seek(length-128) se leen directamente.

06
Audio WAV
javax.sound.sampled
Clip AudioSystem
Javadoc 
import javax.sound.sampled.*;

try (AudioInputStream audio =
        AudioSystem.getAudioInputStream(new File("audio.wav"))) {
    AudioFormat fmt = audio.getFormat();
    System.out.println("SR: " + fmt.getSampleRate() + " Hz");
    Clip clip = AudioSystem.getClip();
    clip.open(audio);
    clip.start();
    Thread.sleep(clip.getMicrosecondLength() / 1000);
    clip.close();
}

07
Buenas
prácticas
Checklist esencial
Rendimiento java.nio
1
Usa siempre try-with-resources para cerrar streams. Nunca uses .close() explícito sin try/finally.
2
Lee en bloques (byte[4096] o más). La diferencia respecto a leer byte a byte puede ser de 50× a 200×.
3
Declara serialVersionUID en todas las clases Serializable. Sin él Java lo genera y puede cambiar al recompilar.
4
Ten en cuenta el endianness del formato. BMP y WAV son little-endian: usa Integer.reverseBytes().
5
Para código moderno usa java.nio.file.Files: readAllBytes(), write() y copy() son más seguros y expresivos.
6
Valida la existencia y permisos del fichero antes de abrirlo y captura IOException con mensajes descriptivos.

ALTERNATIVA MODERNA · java.nio.file

import java.nio.file.Files;
import java.nio.file.Path;

byte[] datos = Files.readAllBytes(Path.of("imagen.bmp"));
Files.write(Path.of("copia.bmp"), datos);
Files.copy(Path.of("origen.bmp"), Path.of("destino.bmp"));
