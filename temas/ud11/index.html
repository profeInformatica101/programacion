Texto vs Binario

Texto: Legible por humanos, fácil de depurar. Ocupa más espacio y parseo lento.
Binario: Secuencia de bytes, más eficiente y compacto, no legible directamente.

Los ficheros binarios son ideales para almacenar grandes volúmenes de datos, imágenes, objetos Java y tipos primitivos de forma rápida.

Streams básicos

FileInputStream y FileOutputStream permiten leer y escribir bytes directamente.

// Ejemplo básico
FileOutputStream fos = new FileOutputStream("datos.bin");
fos.write(65);  // escribe el byte 'A'
fos.close();
Siempre cerrar flujos o usar try-with-resources.

Tipos primitivos

DataInputStream / DataOutputStream

Permiten guardar int, double, boolean, float, long de forma portátil y estructurada.

DataOutputStream dos = new DataOutputStream(new FileOutputStream("primitivos.bin"));
dos.writeInt(42);
dos.writeDouble(3.1416);
dos.writeBoolean(true);
dos.close();
IMPORTANTE: Respetar el orden exacto al leer: writeInt → readInt, writeDouble → readDouble.

Serialización de objetos

import java.io.Serializable;

class Alumno implements Serializable {
    private static final long serialVersionUID = 1L;
    String nombre;
    int nota;
    public Alumno(String nombre, int nota) {
        this.nombre = nombre;
        this.nota = nota;
    }
    @Override
    public String toString() { return nombre + ": " + nota; }
}
// Escritura
ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("alumnos.ser"));
oos.writeObject(new Alumno("Lucía", 9));
oos.close();

// Lectura
ObjectInputStream ois = new ObjectInputStream(new FileInputStream("alumnos.ser"));
Alumno a = (Alumno) ois.readObject();
System.out.println(a);
Consejo: Los objetos deben implementar Serializable y definir serialVersionUID.
