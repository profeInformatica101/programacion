Gestión de datos binarios en Java: flujos, serialización y acceso directo

FileInputStream / FileOutputStream ObjectInputStream / ObjectOutputStream RandomAccessFile Serialización try-with-resources Endianness
1
Introducción

Todos los ficheros son secuencias de bytes, pero podemos distinguir:

  • Texto: interpretados como caracteres (ASCII / UTF).
  • Binarios: formatos específicos (imagen, audio, datos de aplicación…).

Para manejar un fichero binario directamente hay que conocer su formato byte a byte. El formato BMP, por ejemplo, tiene una cabecera estructurada:

OffsetBytesContenido
0–12Identificador del formato ("BM")
2–54Tamaño del fichero (int, Little Endian)
18–214Anchura en píxeles (int, Little Endian)
22–254Altura en píxeles (int, Little Endian)
💡 Como programadores podemos diseñar nuestros propios formatos binarios. Para formatos estándar existen bibliotecas como javax.imageio que simplifican la lectura y escritura de imágenes.
2
Acceso secuencial — Streams

FileInputStream y FileOutputStream son subclases de las clases abstractas InputStream / OutputStream del paquete java.io.

Lectura con FileInputStream

read() devuelve el byte leído como int (8 bits útiles, resto a cero) o -1 al llegar al EOF.

Java
FileInputStream entrada = null;
try {
    entrada = new FileInputStream("fichero.bin");
} catch (FileNotFoundException e) {
    System.err.println("Error al abrir el fichero");
}

if (entrada != null) {
    try {
        while (entrada.available() > 0) {
            String hex = Integer.toHexString(entrada.read());
            if (hex.length() == 1) System.out.print("0");
            System.out.print(hex + " ");
        }
        entrada.close();
    } catch (IOException e) {
        System.err.println("Error al leer el fichero");
    }
}
Escritura con FileOutputStream

El constructor acepta un segundo argumento append: true añade al final; false (por defecto) sobreescribe. write() solo escribe el byte menos significativo del int.

Java
FileOutputStream salida = null;
try {
    salida = new FileOutputStream("nuevo_fichero.bin");
} catch (FileNotFoundException e) {
    System.err.println("Error al abrir el fichero");
}

if (salida != null) {
    try {
        for (int i = 0; i < 256; i++) {
            salida.write(i);
        }
        salida.close();
    } catch (IOException e) {
        System.err.println("Error al escribir fichero");
    }
}
⚠️ Siempre cerrar el fichero con close() para vaciar el buffer y evitar pérdida de datos. Preferir try-with-resources.
Leer un array de bytes de golpe
Java
byte[] buffer = new byte[1024];
try (FileInputStream entrada = new FileInputStream("datos.bin")) {
    int bytesLeidos = entrada.read(buffer);
    System.out.println("Bytes leídos: " + bytesLeidos);
}
Copiar un fichero binario
Java
try (FileInputStream  origen  = new FileInputStream("original.bin");
     FileOutputStream destino = new FileOutputStream("copia.bin")) {
    int dato;
    while ((dato = origen.read()) != -1) {
        destino.write(dato);
    }
    System.out.println("Copia completada.");
} catch (IOException e) {
    System.err.println("Error: " + e.getMessage());
}
3
Serialización de objetos

Permite guardar objetos Java completos en un fichero y recuperarlos en sesiones posteriores de forma transparente.

Requisitos de una clase serializable
  • Implementar la interfaz Serializable (solo declarativa, sin métodos)
  • Tener un constructor por defecto (sin parámetros)
  • Declarar serialVersionUID — evita InvalidClassException entre versiones
  • Marcar con transient los atributos que no se quieran persistir
Clase MiEmpresa — ejemplo completo
Java
public class MiEmpresa implements Serializable {
    private static final long serialVersionUID = 6368263803L;

    private List<Cliente>   clientes;
    private List<Empleado>  empleados;
    private List<Producto>  productos;
    private double capital;
    private String nombre;

    public MiEmpresa() { // constructor por defecto obligatorio
        clientes  = new ArrayList<>();
        empleados = new ArrayList<>();
        productos = new ArrayList<>();
        capital   = 100000;
        nombre    = "Mi Empresa";
    }
}
Guardar objeto
Java
public static void guardarDatos(MiEmpresa me) {
    try (ObjectOutputStream salida = new ObjectOutputStream(
            new FileOutputStream("mi_empresa.dat"))) {
        salida.writeObject(me);
    } catch (Exception e) {
        System.err.println("Error al guardar: " + e.getMessage());
    }
}
Cargar objeto
Java
public static MiEmpresa cargarDatos() {
    MiEmpresa me;
    try (ObjectInputStream entrada = new ObjectInputStream(
            new FileInputStream("mi_empresa.dat"))) {
        me = (MiEmpresa) entrada.readObject();
    } catch (Exception e) {
        System.err.println(e);
        me = new MiEmpresa(); // primera ejecución: fichero no existe aún
    }
    return me;
}
Tipos primitivos — mismo orden de escritura / lectura
Java — escritura
try (ObjectOutputStream out = new ObjectOutputStream(
        new FileOutputStream("datos.dat"))) {
    out.writeInt(42);
    out.writeDouble(3.14);
    out.writeBoolean(true);
}
Java — lectura (mismo orden)
try (ObjectInputStream in = new ObjectInputStream(
        new FileInputStream("datos.dat"))) {
    int     num  = in.readInt();
    double  pi   = in.readDouble();
    boolean flag = in.readBoolean();
    System.out.println(num + " / " + pi + " / " + flag);
}
🚫
Inconvenientes: el formato es propio de Java (no compatible con otros lenguajes). Si se modifica la estructura de la clase puede romperse la compatibilidad con ficheros anteriores. Además, Java usa Big Endian; las plataformas Intel usan Little Endian.

Acceso directo — RandomAccessFile

Permite acceder a cualquier posición del fichero sin leerlo completo, como si fuera un array de bytes con un puntero interno (filePointer).

Modos de apertura
ModoDescripción
"r"Solo lectura
"rw"Lectura y escritura
"rws"Lectura/escritura + sincronización inmediata (datos y metadatos)
"rwd"Lectura/escritura + sincronización solo de datos (mayor rendimiento)
Leer dimensiones de un BMP
Java
try (RandomAccessFile fichero = new RandomAccessFile("imagen.bmp", "r")) {
    fichero.seek(18); // posición 18: anchura (4 bytes, Little Endian)
    int ancho = Integer.reverseBytes(fichero.readInt());
    int alto  = Integer.reverseBytes(fichero.readInt()); // posición 22
    System.out.println("Imagen: " + ancho + " x " + alto + " píxeles");
} catch (IOException e) {
    System.err.println("Error: " + e.getMessage());
}
Modificar la altura de un BMP
Java
int nuevaAltura = 50;
try (RandomAccessFile fichero = new RandomAccessFile("imagen.bmp", "rw")) {
    fichero.seek(22);
    fichero.writeInt(Integer.reverseBytes(nuevaAltura));
    System.out.println("Altura modificada a " + nuevaAltura + " px.");
} catch (IOException e) {
    System.err.println("Error: " + e.getMessage());
}
Registros de longitud fija

Guardando registros de tamaño fijo podemos acceder directamente al registro N: seek(N * TAM_REGISTRO).

Java
final int TAM_REGISTRO = 4 + 8; // int (4B) + double (8B) = 12 bytes

try (RandomAccessFile raf = new RandomAccessFile("registros.dat", "rw")) {
    // Escribir 5 registros
    for (int i = 0; i < 5; i++) {
        raf.writeInt(i);
        raf.writeDouble(i * 1.5);
    }
    // Leer el registro 3 directamente
    int numRegistro = 3;
    raf.seek((long) numRegistro * TAM_REGISTRO);
    int    id    = raf.readInt();
    double valor = raf.readDouble();
    System.out.println("Registro 3: id=" + id + ", valor=" + valor);
}
ℹ️ seek(pos) mueve el puntero · getFilePointer() consulta la posición actual · length() devuelve el tamaño del fichero

Errores comunes y cómo evitarlos NUEVO
ErrorCausaSolución
No cerrar el fichero Buffer de escritura no se vacía; pérdida de datos Usar siempre close() o try-with-resources
Sin Serializable NotSerializableException al intentar guardar Implementar Serializable en la clase y en todas sus clases miembro
Sin serialVersionUID InvalidClassException al cambiar la clase Declarar siempre private static final long serialVersionUID = ...L;
Orden de lectura distinto Datos incorrectos o excepción Mantener el mismo orden de lectura y escritura; documentarlo
Ignorar Endianness Número leído incorrecto en ficheros externos Usar Integer.reverseBytes() al leer enteros de ficheros no Java
ClassNotFoundException No capturada al usar readObject() Capturar tanto IOException como ClassNotFoundException
available() en red Puede devolver 0 aunque queden datos En redes usar read() != -1 en vez de available()
Patrón recomendado — try-with-resources

Desde Java 7 el fichero se cierra automáticamente aunque haya excepción:

Java
try (FileInputStream entrada = new FileInputStream("datos.bin")) {
    int dato;
    while ((dato = entrada.read()) != -1) {
        System.out.print(Integer.toHexString(dato) + " ");
    }
} catch (IOException e) {
    System.err.println("Error: " + e.getMessage());
}
// entrada.close() se llama automáticamente

1
Contar bytes de un fichero

Escribe un programa que abra un fichero binario cualquiera e imprima cuántos bytes tiene.

Java
import java.io.*;

public class ContarBytes {
    public static void main(String[] args) {
        try (FileInputStream entrada = new FileInputStream("fichero.bin")) {
            int contador = 0;
            while (entrada.read() != -1) contador++;
            System.out.println("El fichero tiene " + contador + " bytes.");
        } catch (IOException e) {
            System.err.println("Error: " + e.getMessage());
        }
    }
}
2
Guardar y recuperar una lista de productos

Crea la clase Producto (id: int, nombre: String, precio: double) serializable. Guarda una lista con 3 productos en productos.dat y vuelve a cargarla imprimiendo cada producto.

Java
import java.io.*;
import java.util.*;

class Producto implements Serializable {
    private static final long serialVersionUID = 1L;
    private int id;
    private String nombre;
    private double precio;

    public Producto() {}
    public Producto(int id, String nombre, double precio) {
        this.id = id; this.nombre = nombre; this.precio = precio;
    }
    public String toString() {
        return id + " | " + nombre + " | " + precio + " EUR";
    }
}

public class GestionProductos {

    public static void guardar(List<Producto> lista) throws Exception {
        try (ObjectOutputStream out = new ObjectOutputStream(
                new FileOutputStream("productos.dat"))) {
            out.writeObject(lista);
        }
    }

    @SuppressWarnings("unchecked")
    public static List<Producto> cargar() throws Exception {
        try (ObjectInputStream in = new ObjectInputStream(
                new FileInputStream("productos.dat"))) {
            return (List<Producto>) in.readObject();
        }
    }

    public static void main(String[] args) throws Exception {
        List<Producto> productos = new ArrayList<>();
        productos.add(new Producto(1, "Teclado",  45.99));
        productos.add(new Producto(2, "Ratón",    19.50));
        productos.add(new Producto(3, "Monitor", 299.00));

        guardar(productos);
        System.out.println("Lista guardada.");

        List<Producto> recuperados = cargar();
        recuperados.forEach(System.out::println);
    }
}

Base de datos simple con RandomAccessFile

Crea un fichero que almacene 5 registros con la estructura id (int, 4B) + puntuación (double, 8B). Después lee el registro número 2 directamente sin leer los anteriores.

Java
import java.io.*;

public class BaseDatosSimple {
    static final int TAM_REGISTRO = 4 + 8; // int + double = 12 bytes

    public static void main(String[] args) throws IOException {
        // ESCRITURA
        try (RandomAccessFile raf = new RandomAccessFile("scores.dat", "rw")) {
            for (int i = 0; i < 5; i++) {
                raf.writeInt(i + 1);
                raf.writeDouble(100.0 - i * 7.5);
            }
        }
        // LECTURA DIRECTA del registro 2 (índice 2)
        try (RandomAccessFile raf = new RandomAccessFile("scores.dat", "r")) {
            int numRegistro = 2;
            raf.seek((long) numRegistro * TAM_REGISTRO);
            int    id    = raf.readInt();
            double score = raf.readDouble();
            System.out.println("Registro 2: id=" + id + ", puntuación=" + score);
        }
    }
}

Verificar cabecera de un fichero BMP

Abre un fichero e imprime "Es un BMP válido" si sus dos primeros bytes son 'B' (66) y 'M' (77). Si no, indica que el formato es incorrecto.

Java
import java.io.*;

public class VerificarBMP {
    public static void main(String[] args) throws IOException {
        try (RandomAccessFile raf = new RandomAccessFile("imagen.bmp", "r")) {
            int b1 = raf.read(); // 'B' = 66
            int b2 = raf.read(); // 'M' = 77
            if (b1 == 'B' && b2 == 'M') {
                System.out.println("Es un BMP válido. Tamaño: "
                    + raf.length() + " bytes.");
            } else {
                System.out.println("Formato incorrecto. Cabecera: "
                    + (char) b1 + (char) b2);
            }
        }
    }
}

Resumen rápido
  • Ficheros binarios = secuencia de bytes sin interpretación de texto
  • FileInputStream / FileOutputStream → acceso secuencial byte a byte
  • ObjectInputStream / ObjectOutputStream → serialización de objetos Java
  • RandomAccessFile → acceso directo con seek()
  • Endianness: Java = Big Endian · Intel = Little Endian → usar Integer.reverseBytes() al leer ficheros externos
  • Usar try-with-resources para cerrar ficheros automáticamente
  • Declarar siempre serialVersionUID en clases serializables
