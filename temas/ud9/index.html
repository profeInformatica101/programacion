Una interfaz en Java define un tipo y un contrato de comportamiento. Se parece a una clase, pero su intención es declarativa: describe qué debe ofrecer una clase, no necesariamente cómo lo hace.

  • Se declara con interface (en un archivo .java).
  • No se puede instanciar (no hay new sobre una interfaz).
  • Sus atributos son constantes (public static final).
  • Sus métodos son, por defecto, abstract y public (con ampliaciones desde Java 8).
  • Una clase puede implementar múltiples interfaces.
Idea clave: implementar una interfaz significa “heredar la obligación de implementar sus métodos”.

Referencia (tutorial oficial): Creating Interfaces (Oracle)

Una interfaz declara métodos (normalmente abstractos). Una clase la implementa con implements y debe proporcionar el código de esos métodos.

Elemento Ejemplo Nota
Declaración de interfaz 
public interface TieneArea {
  double getArea();
}
 Método abstracto (contrato).
Clase que implementa 
public class Circulo implements TieneArea {
  private double radio;

  public Circulo(double radio){ this.radio = radio; }

  @Override
  public double getArea(){
    return Math.PI * radio * radio;
  }
}
 Si no implementa, debe ser abstract.
Implementación múltiple 
class X implements A, B, C { /* ... */ }
 Una clase puede implementar tantas interfaces como necesite.
Ojo: “Implementar” no es “heredar de una clase”. Las interfaces definen tipos y contratos, no un estado compartido.

Una interfaz puede extender otras interfaces con extends. Esta herencia también es múltiple. 

public interface A { }
public interface B { }
public interface C extends A, B { }
Resultado: C hereda los miembros (métodos y constantes) de A y B.

Además de métodos abstractos, desde Java 8 las interfaces pueden tener:

  • abstract contrato obligatorio.
  • default método con implementación heredable.
  • static método utilitario propio de la interfaz.
Método default

Útil para mantener compatibilidad en APIs: se añade funcionalidad sin obligar a reescribir todas las clases que ya implementaban la interfaz. 

public interface Coloreado {
  Color getColor();

  default Color getColorInverso() {
    Color c = getColor();
    return new Color(255 - c.getRed(),
                     255 - c.getGreen(),
                     255 - c.getBlue());
  }
}
Método static

Se llama sobre la propia interfaz (no sobre objetos). 

public interface Utilidades {
  static void saludar(){
    System.out.println("Hola");
  }
}

// Uso:
Utilidades.saludar();

Referencias oficiales: Default Methods (Oracle)

Una interfaz funcional tiene un único método abstracto. Se suele marcar con @FunctionalInterface. 

@FunctionalInterface
public interface Operacion {
  int aplicar(int a, int b);
}

Esto permite usar lambdas: 

Operacion suma = (a, b) -> a + b;
System.out.println(suma.aplicar(2, 3)); // 5
En la librería estándar hay muchas interfaces funcionales en java.util.function (por ejemplo Predicate, Function, Consumer).

Comparable<T> define el orden natural de una clase, mediante el método compareTo(T o).

Usa genéricos: Comparable<Persona> (evita casts y errores). 
public class Persona implements Comparable<Persona> {
  private int edad;

  @Override
  public int compareTo(Persona otra) {
    return this.edad - otra.edad; // orden por edad
  }
}

Convención del resultado:

  • negativo → this va antes
  • positivo → this va después
  • cero → equivalentes

Se usa en ordenaciones como: Arrays.sort() y Collections.sort().

Referencia oficial: Comparable (Java SE 8)

Comparator<T> permite definir órdenes alternativos sin modificar la clase.

Comparator con lambda 
Comparator<Persona> porNombre =
  (p1, p2) -> p1.getNombre()
                .compareTo(p2.getNombre());
Comparator.comparing (Java 8+) 
Comparator<Persona> porEdad =
  Comparator.comparing(Persona::getEdad);

Comparator<Persona> porEdadDesc =
  Comparator.comparing(Persona::getEdad)
            .reversed();

Para ordenar: 

Arrays.sort(personas, porNombre);
Collections.sort(lista, porEdadDesc);

Referencia oficial: Comparator (Java SE 8)

Ideas clave
  • Una interfaz define un contrato.
  • No se instancia; se implementa.
  • Una clase puede implementar múltiples interfaces.
  • default y static existen desde Java 8.
  • Comparable → orden natural.
  • Comparator → orden alternativo.
