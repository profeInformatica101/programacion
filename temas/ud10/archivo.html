Introducción

Primeros pasos sólidos para persistir datos con Java sin necesidad de base de datos.

Un archivo de texto plano es un fichero que almacena información en forma de caracteres legibles, normalmente codificados en UTF-8 o similares. Su extensión más habitual es .txt, aunque también pueden usarse otros formatos de texto simples como .csv o .log.

En programación se utilizan para guardar información de manera sencilla: configuraciones, listados, registros, exportaciones, pequeños sistemas persistentes o pruebas iniciales antes de pasar a soluciones más complejas.

¿Por qué empezar por archivos de texto?

Porque permiten entender el flujo completo de persistencia: el programa crea datos, los transforma a texto, los guarda, los recupera y reconstruye la información posteriormente.

Simplicidad

No requieren instalación de servicios externos ni configuración compleja.

Legibilidad

Se pueden abrir con un editor cualquiera y revisar su contenido fácilmente.

Didáctica

Ayudan a comprender entrada, salida, formato y recuperación de datos.

Conceptos básicos

Clases esenciales de Java para trabajar con archivos de texto.

¿Qué es un archivo .txt?

Es un fichero que contiene texto sin formato enriquecido. No guarda colores, estilos o estructuras avanzadas como un documento de Word. Esto lo hace ideal para almacenar líneas simples, registros separados por delimitadores o información temporal.

Clase Función principal Uso típico
File Representa un archivo o directorio del sistema Comprobar existencia, crear rutas, consultar propiedades
FileWriter Escribe caracteres en un archivo Guardar texto simple o sobrescribir contenido
BufferedWriter Escritura más eficiente mediante buffer Escribir múltiples líneas con mayor comodidad
FileReader Lee caracteres desde un archivo Lecturas básicas
BufferedReader Permite leer línea a línea Procesar registros de texto de forma cómoda

Enlaces a la API oficial de Java

Documentación oficial de Oracle para Java 17 de las clases utilizadas en este tutorial:

File — representación de rutas y archivos FileWriter — escritura de caracteres en archivo BufferedWriter — escritura con buffer FileReader — lectura de caracteres desde archivo BufferedReader — lectura línea a línea
Consejo importante: para la mayoría de ejercicios académicos y pequeños proyectos, BufferedReader y BufferedWriter resultan más cómodos que trabajar directamente carácter a carácter.

Ejemplo 1: Crear y escribir en un archivo

Primer escenario: generar un archivo y guardar varias líneas de texto.

En este ejemplo vamos a crear un archivo llamado datos.txt y escribir dentro varias líneas. Se utiliza BufferedWriter junto con FileWriter para mejorar claridad y rendimiento.

Paso a paso

  1. Se crea un objeto de escritura asociado al archivo.
  2. Se escriben varias líneas con write().
  3. Se usa newLine() para separar registros.
  4. Al terminar, el recurso se cierra automáticamente gracias a try-with-resources.
Ejemplo1CrearArchivo.java 
import java.io.BufferedWriter;
import java.io.FileWriter;
import java.io.IOException;

public class Ejemplo1CrearArchivo {
    public static void main(String[] args) {
        // try-with-resources cierra automáticamente el writer
        try (BufferedWriter writer = new BufferedWriter(new FileWriter("datos.txt"))) {

            // Escribimos varias líneas en el archivo
            writer.write("Java");
            writer.newLine();
            writer.write("Archivos de texto");
            writer.newLine();
            writer.write("DAW y DAM");

            System.out.println("Archivo creado y datos escritos correctamente.");

        } catch (IOException e) {
            System.out.println("Error al escribir el archivo: " + e.getMessage());
        }
    }
}
Observación: si el archivo no existe, FileWriter lo crea. Si ya existe, por defecto sobrescribe su contenido.

Ejemplo 2: Leer datos de un archivo

Lectura línea a línea para procesar información textual de forma limpia.

Una de las operaciones más habituales es recuperar el contenido del archivo. Para ello, BufferedReader permite leer línea por línea mediante readLine().

Ejemplo2LeerArchivo.java 
import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class Ejemplo2LeerArchivo {
    public static void main(String[] args) {
        try (BufferedReader reader = new BufferedReader(new FileReader("datos.txt"))) {

            String linea;

            // Leemos hasta que no queden más líneas
            while ((linea = reader.readLine()) != null) {
                System.out.println("Línea leída: " + linea);
            }

        } catch (IOException e) {
            System.out.println("Error al leer el archivo: " + e.getMessage());
        }
    }
}

¿Qué ocurre aquí?

  • FileReader abre el archivo para lectura.
  • BufferedReader añade un buffer para leer con más comodidad.
  • readLine() devuelve una línea completa o null si se alcanzó el final.

Ejemplo 3: Guardar objetos en un archivo

Pasamos de texto suelto a persistencia estructurada de objetos.

Un uso muy común consiste en guardar objetos como líneas de texto. Para ello, definimos una clase Tarea y convertimos cada objeto a una línea con un formato delimitado por punto y coma:

Formato elegido: id;titulo;modulo;estado
Tarea.java 
public class Tarea {
    private int id;
    private String titulo;
    private String modulo;
    private String estado;

    public Tarea(int id, String titulo, String modulo, String estado) {
        this.id = id;
        this.titulo = titulo;
        this.modulo = modulo;
        this.estado = estado;
    }

    public int getId() {
        return id;
    }

    public String getTitulo() {
        return titulo;
    }

    public String getModulo() {
        return modulo;
    }

    public String getEstado() {
        return estado;
    }

    // Convierte el objeto a una línea de texto lista para guardar
    public String toLinea() {
        return id + ";" + titulo + ";" + modulo + ";" + estado;
    }
}
GuardarTareas.java 
import java.io.BufferedWriter;
import java.io.FileWriter;
import java.io.IOException;

public class GuardarTareas {
    public static void main(String[] args) {
        Tarea t1 = new Tarea(1, "Crear formulario", "Programación", "pendiente");
        Tarea t2 = new Tarea(2, "Leer archivo", "Entornos", "hecha");

        try (BufferedWriter writer = new BufferedWriter(new FileWriter("tareas.txt"))) {
            writer.write(t1.toLinea());
            writer.newLine();
            writer.write(t2.toLinea());
            writer.newLine();

            System.out.println("Tareas guardadas correctamente.");
        } catch (IOException e) {
            System.out.println("Error al guardar tareas: " + e.getMessage());
        }
    }
}

Este enfoque es muy útil porque separa claramente la lógica del objeto y la forma en la que se almacena.

Ejemplo 4: Leer archivo y reconstruir objetos

Ahora hacemos el camino inverso: de línea de texto a objeto Java.

Cuando leemos cada línea, podemos dividirla con split(";") y crear una instancia de Tarea con esos valores.

LeerTareas.java 
import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class LeerTareas {
    public static void main(String[] args) {
        try (BufferedReader reader = new BufferedReader(new FileReader("tareas.txt"))) {
            String linea;

            while ((linea = reader.readLine()) != null) {
                String[] partes = linea.split(";");

                int id = Integer.parseInt(partes[0]);
                String titulo = partes[1];
                String modulo = partes[2];
                String estado = partes[3];

                Tarea tarea = new Tarea(id, titulo, modulo, estado);

                System.out.println("ID: " + tarea.getId());
                System.out.println("Título: " + tarea.getTitulo());
                System.out.println("Módulo: " + tarea.getModulo());
                System.out.println("Estado: " + tarea.getEstado());
                System.out.println("--------------------------");
            }

        } catch (IOException e) {
            System.out.println("Error al leer tareas: " + e.getMessage());
        }
    }
}
Precaución: este formato funciona bien si los datos no contienen el delimitador ;. En sistemas más grandes conviene validar o usar formatos más robustos como CSV bien tratado, JSON o base de datos.

Mini Proyecto: Sistema simple de tareas

Aplicación básica orientada a consolidar escritura, lectura y reconstrucción de objetos.

El proyecto consiste en un pequeño sistema por consola que utiliza un archivo tareas.txt como almacenamiento persistente. Al arrancar, se leen las tareas ya guardadas. Durante la ejecución, el usuario puede añadir nuevas tareas y mostrarlas. En esta versión no se usan colecciones, sino un array de objetos y un contador, porque es una solución más adecuada para alumnado que todavía no ha trabajado con List o ArrayList.

Enfoque didáctico: este mini proyecto está resuelto únicamente con arrays, posiciones y un contador, para adaptarse a un momento del curso en el que todavía no se han trabajado las colecciones.
1. Añadir tarea

Recoger datos del usuario y crear el objeto Tarea.

2. Mostrar tareas

Recorrer el array en memoria y enseñar su contenido.

3. Guardar en archivo

Convertir cada tarea en una línea y persistirla.

4. Leer al iniciar

Reconstruir el estado previo desde tareas.txt.

GestorTareas.java 
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.File;
import java.io.FileReader;
import java.io.FileWriter;
import java.io.IOException;
import java.util.Scanner;

public class GestorTareas {

    private static final String NOMBRE_ARCHIVO = "tareas.txt";
    private static final int CAPACIDAD_MAXIMA = 100;
    private static final Tarea[] tareas = new Tarea[CAPACIDAD_MAXIMA];
    private static int totalTareas = 0;

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        cargarTareas();

        int opcion;
        do {
            System.out.println("\n=== MENÚ DE TAREAS ===");
            System.out.println("1. Añadir tarea");
            System.out.println("2. Mostrar tareas");
            System.out.println("3. Guardar tareas");
            System.out.println("0. Salir");
            System.out.print("Elige una opción: ");
            opcion = Integer.parseInt(sc.nextLine());

            switch (opcion) {
                case 1:
                    anadirTarea(sc);
                    break;
                case 2:
                    mostrarTareas();
                    break;
                case 3:
                    guardarTareas();
                    break;
                case 0:
                    guardarTareas();
                    System.out.println("Saliendo del programa...");
                    break;
                default:
                    System.out.println("Opción no válida.");
            }
        } while (opcion != 0);

        sc.close();
    }

    private static void anadirTarea(Scanner sc) {
        if (totalTareas >= CAPACIDAD_MAXIMA) {
            System.out.println("No se pueden añadir más tareas. Array lleno.");
            return;
        }

        System.out.print("ID: ");
        int id = Integer.parseInt(sc.nextLine());

        System.out.print("Título: ");
        String titulo = sc.nextLine();

        System.out.print("Módulo: ");
        String modulo = sc.nextLine();

        System.out.print("Estado: ");
        String estado = sc.nextLine();

        tareas[totalTareas] = new Tarea(id, titulo, modulo, estado);
        totalTareas++;
        System.out.println("Tarea añadida correctamente.");
    }

    private static void mostrarTareas() {
        if (totalTareas == 0) {
            System.out.println("No hay tareas registradas.");
            return;
        }

        for (int i = 0; i < totalTareas; i++) {
            System.out.println(tareas[i].toLinea());
        }
    }

    private static void guardarTareas() {
        try (BufferedWriter writer = new BufferedWriter(new FileWriter(NOMBRE_ARCHIVO))) {
            for (int i = 0; i < totalTareas; i++) {
                writer.write(tareas[i].toLinea());
                writer.newLine();
            }
            System.out.println("Tareas guardadas en archivo.");
        } catch (IOException e) {
            System.out.println("Error al guardar: " + e.getMessage());
        }
    }

    private static void cargarTareas() {
        File archivo = new File(NOMBRE_ARCHIVO);

        if (!archivo.exists()) {
            System.out.println("No existe archivo previo. Se iniciará vacío.");
            return;
        }

        try (BufferedReader reader = new BufferedReader(new FileReader(archivo))) {
            String linea;
            while ((linea = reader.readLine()) != null && totalTareas < CAPACIDAD_MAXIMA) {
                String[] partes = linea.split(";");
                int id = Integer.parseInt(partes[0]);
                String titulo = partes[1];
                String modulo = partes[2];
                String estado = partes[3];

                tareas[totalTareas] = new Tarea(id, titulo, modulo, estado);
                totalTareas++;
            }
        } catch (IOException e) {
            System.out.println("Error al cargar tareas: " + e.getMessage());
        }
    }
}

Qué conceptos consolida este mini proyecto

  • Uso de un array de objetos y un contador para gestionar los elementos en memoria.
  • Persistencia simple basada en archivo de texto.
  • Separación entre lectura inicial, gestión de datos y guardado final.
  • Formato de serialización manual mediante delimitadores.

Buenas prácticas

La diferencia entre un ejemplo funcional y un código profesional suele estar aquí.

Manejo de excepciones

Captura IOException y comunica errores útiles al usuario o al desarrollador.

Cerrar archivos

Nunca dejes recursos abiertos. Puede provocar bloqueos o pérdida de datos.

Try-with-resources

Es la forma moderna, segura y recomendada para gestionar streams y readers/writers.

Recomendación docente y profesional

Empieza con archivos de texto para comprender el mecanismo. Después evoluciona hacia clases utilitarias, validación de datos, separación por capas y, cuando el proyecto lo requiera, persistencia con base de datos.

Resumen rápido

  • Un archivo .txt guarda texto plano.
  • FileWriter y BufferedWriter permiten escribir.
  • FileReader y BufferedReader permiten leer.
  • Los objetos pueden convertirse a líneas de texto.
  • Con split(";") se reconstruyen objetos.

Persistir en archivos de texto no solo sirve para guardar datos: también enseña a diseñar formatos, validar información y mantener consistencia entre memoria y almacenamiento. Además, este enfoque con arrays permite reforzar contenidos previos antes de dar el salto a las colecciones.

Crear el proyecto con Maven

Antes de programar, conviene generar una base de proyecto limpia y profesional.

Una forma recomendada de comenzar es crear el proyecto con Maven, ya que permite organizar el código, gestionar dependencias y mantener una estructura estándar. Con el siguiente comando se genera un proyecto Java listo para empezar a trabajar:

Ventaja: usar Maven desde el inicio ayuda al alumnado a familiarizarse con una estructura profesional de proyecto, especialmente útil en DAW y DAM.
Comando Maven 
mvn archetype:generate \
  -DarchetypeGroupId=org.apache.maven.archetypes \
  -DarchetypeArtifactId=maven-archetype-quickstart \
  -DarchetypeVersion=1.5 \
  -DgroupId=es.iescamas \
  -DartifactId=Archivos \
  -Dversion=1.0.0-SNAPSHOT \
  -DjavaCompilerVersion=17 \
  -DjunitVersion=5.11.0 \
  -DinteractiveMode=false

¿Qué hace este comando?

Parámetro Significado
-DarchetypeGroupId Indica el grupo del arquetipo Maven que se va a usar como plantilla.
-DarchetypeArtifactId Selecciona la plantilla concreta del proyecto, en este caso maven-archetype-quickstart.
-DarchetypeVersion Fija la versión del arquetipo.
-DgroupId Define el paquete base del proyecto, aquí es.iescamas.
-DartifactId Nombre del proyecto Maven: Archivos.
-Dversion Versión inicial del proyecto.
-DjavaCompilerVersion Configura Java 17 como versión de compilación.
-DjunitVersion Establece la versión de JUnit para pruebas.
-DinteractiveMode=false Evita preguntas interactivas y genera el proyecto directamente.

Estructura generada

Después de ejecutar el comando, Maven creará una carpeta llamada Archivos con una estructura muy parecida a esta:

Recomendación: una vez generado el proyecto, puedes renombrar App.java por una clase más representativa, como GestorTareas.java, y empezar a organizar mejor el código según el dominio del ejercicio.
