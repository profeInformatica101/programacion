Tarea práctica: GitHub + Java

Guía rápida y ejecutable para que los alumnos aprendan fork, SSH, clonación, commits y subir un proyecto Java.

Repositorio base: profeInformatica101/ejerciciosJAVA

1) Fork del repositorio

  1. Ir al repositorio: https://github.com/profeInformatica101/ejerciciosJAVA
  2. Hacer clic en Fork (arriba a la derecha). Se creará una copia en tu cuenta.
  3. Verificar que el repo aparece en tu perfil de GitHub.

2) Configurar claves SSH y clonar

Generar clave SSH en local (ejemplo con ed25519) y añadir la clave pública en GitHub → Settings → SSH and GPG keys.

Generar claves SSH 
ssh-keygen -t ed25519 -C "tu_email@ejemplo.com"
# Si pides ruta por defecto, pulsa Enter
# Luego copia la clave pública (~/.ssh/id_ed25519.pub) y pégala en GitHub
Clonar el fork (usando SSH) 
git clone git@github.com:TU_USUARIO/ejerciciosJAVA.git
cd ejerciciosJAVA

3) Editar README (añadir la chuleta de comandos)

Editar README.md en la raíz del repositorio para incluir la mini-chuleta con los comandos básicos.

### Chuleta Git

- `git status` → Ver estado del repositorio
- `git add .` → Añadir cambios
- `git commit -m "mensaje"` → Guardar cambios localmente
- `git push` → Subir cambios al remoto
- `git pull` → Traer cambios del remoto

Commit y push de ejemplo:

git add README.md
git commit -m "Añadida chuleta de comandos Git"
git push origin main

4) Crear un proyecto Java sencillo

Dentro del repo crear la carpeta MiProyectoJava y añadir un Main.java:

// MiProyectoJava/Main.java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hola GitHub desde Java!");
    }
}

Añadir, commitear y pushear:

git add MiProyectoJava/Main.java
git commit -m "Primer proyecto Java añadido"
git push origin main

5) Comprobación final

  • Mirar en GitHub: comprobar que el README.md está actualizado.
  • Comprobar que existe la carpeta MiProyectoJava con el Main.java.
  • Pedir a los alumnos que compartan el enlace a su repo fork como entregable.

Consejos para el/la profesor(a)

  • Revisar que los alumnos han añadido la clave SSH: en la máquina local pueden probar ssh -T git@github.com.
  • Si usan Windows y no tienen WSL, pueden seguir el cliente de Git para Windows o usar Git Bash.
  • Evaluación sencilla: 1 punto por README actualizado, 1 punto por proyecto Java subido, 1 punto por SSH funcionado (captura de pantalla o salida del comando).
