Conceptos clave antes de programar

JDBC (Java DataBase Connectivity) es la API estándar de Java para conectar aplicaciones con bases de datos relacionales. Forma parte del JDK (java.sql.*) y actúa como una capa de abstracción entre tu código y el SGBD concreto.

La arquitectura JDBC tiene cuatro capas:

  1. Aplicación Java — código de negocio que usa la API.
  2. JDBC API — interfaces estándar (Connection, Statement, ResultSet…).
  3. Driver JDBC — implementación específica del fabricante (SQLite, MySQL, PostgreSQL…).
  4. SGBD — el servidor de base de datos real.
Si cambias de SQLite a MySQL solo tienes que cambiar la URL de conexión y el JAR del driver. El resto del código JDBC no cambia.

La URL de conexión identifica unívocamente el SGBD, el host, el puerto y la base de datos:

SGBDURL JDBC
SQLitejdbc:sqlite:ruta/fichero.db
MySQLjdbc:mysql://localhost:3306/nombre_bd
PostgreSQLjdbc:postgresql://localhost:5432/nombre_bd
H2 (memoria)jdbc:h2:mem:nombre_bd

Desde JDBC 4.0 (Java 6+) el driver se carga automáticamente si el JAR está en el classpath. Ya no es necesario llamar a Class.forName("...").

SQL Injection ocurre cuando un atacante inyecta código SQL malicioso en un parámetro de entrada. Ejemplo vulnerable:

⚠ Código VULNERABLE — no usar 
// Si login = "' OR '1'='1" → acceso universal
String sql = "SELECT * FROM usuarios WHERE login='"
             + login + "' AND pass='" + pass + "'";
Statement st = conn.createStatement();
ResultSet rs = st.executeQuery(sql); // PELIGROSO

Solución: usa siempre PreparedStatement con marcadores de posición ?. El driver escapa los valores automáticamente.

Regla de oro: Ningún valor de usuario debe concatenarse directamente en una cadena SQL. Siempre PreparedStatement.
Ejemplos de código comentados
01 — Establecer conexión con SQLite (try-with-resources)

El bloque try-with-resources garantiza que los recursos se cierran automáticamente aunque se lance una excepción.

ConexionBD.java 
import java.sql.*;

public class ConexionBD {

    private static final String URL = "jdbc:sqlite:instituto.db";

    /** Devuelve una nueva conexión activa al SGBD */
    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL);
    }

    // Ejemplo de uso:
    public static void main(String[] args) {
        try (Connection conn = getConnection()) {
            System.out.println("Conectado: " + conn.getMetaData().getDatabaseProductName());
        } catch (SQLException e) {
            System.err.println("Error al conectar: " + e.getMessage());
        }
    }
}
02 — Crear tabla y operaciones CRUD con PreparedStatement

Operaciones CREATE TABLE, INSERT, UPDATE y DELETE seguras con sentencias parametrizadas.

AlumnoDAOImpl.java — fragmentos CRUD 
// ── DDL: crear tabla si no existe ─────────────────────────
String ddl = """
    CREATE TABLE IF NOT EXISTS alumnos (
        id     INTEGER PRIMARY KEY AUTOINCREMENT,
        nombre TEXT    NOT NULL,
        ciclo  TEXT    NOT NULL,
        nota   REAL    DEFAULT 0
    )""";
conn.createStatement().execute(ddl);

// ── CREATE (INSERT) ────────────────────────────────────────
String sqlIns = "INSERT INTO alumnos(nombre, ciclo, nota) VALUES(?, ?, ?)";
try (PreparedStatement ps = conn.prepareStatement(sqlIns)) {
    ps.setString(1, "Laura García");
    ps.setString(2, "DAW");
    ps.setDouble(3, 8.5);
    ps.executeUpdate();
}

// ── UPDATE ─────────────────────────────────────────────────
String sqlUpd = "UPDATE alumnos SET nota = ? WHERE id = ?";
try (PreparedStatement ps = conn.prepareStatement(sqlUpd)) {
    ps.setDouble(1, 9.2);
    ps.setInt(2, 1);
    int filas = ps.executeUpdate();   // filas afectadas
    System.out.println("Filas actualizadas: " + filas);
}

// ── DELETE ─────────────────────────────────────────────────
String sqlDel = "DELETE FROM alumnos WHERE id = ?";
try (PreparedStatement ps = conn.prepareStatement(sqlDel)) {
    ps.setInt(1, 3);
    ps.executeUpdate();
}
03 — Consulta SELECT y mapeo ORM al objeto Alumno

Recorrido del ResultSet y construcción de objetos Java a partir de las filas devueltas.

AlumnoDAOImpl.java — método listarPorCiclo() 
public List<Alumno> listarPorCiclo(String ciclo) {
    List<Alumno> lista = new ArrayList<>();
    String sql = "SELECT id, nombre, ciclo, nota FROM alumnos WHERE ciclo = ? ORDER BY nombre";

    try (Connection        conn = ConexionBD.getConnection();
         PreparedStatement ps   = conn.prepareStatement(sql)) {

        ps.setString(1, ciclo);

        try (ResultSet rs = ps.executeQuery()) {
            while (rs.next()) {
                // Mapeo fila → objeto (ORM manual)
                Alumno a = new Alumno();
                a.setId(rs.getInt("id"));
                a.setNombre(rs.getString("nombre"));
                a.setCiclo(rs.getString("ciclo"));
                a.setNota(rs.getDouble("nota"));
                lista.add(a);
            }
        }
    } catch (SQLException e) {
        e.printStackTrace();
    }
    return lista;
}
Buena práctica: abre la conexión en el bloque try-with-resources, no como atributo de instancia. Cada operación abre y cierra su propia conexión para evitar fugas de recursos.

Arquitectura: Patrón DAO

El patrón DAO (Data Access Object) aísla la lógica de acceso a datos del resto de la aplicación. Cambiar de SGBD solo requiere modificar la capa DAO, sin tocar la lógica de negocio.

Modelo
Alumno.java — Entidad POJO Clase Java pura con atributos privados, constructor, getters y setters. Representa una fila de la tabla. Sin dependencias de JDBC.
Interfaz
AlumnoDAO.java — Contrato Define los métodos: insertar(Alumno a), listarTodos(), obtenerPorId(int id), actualizar(Alumno a), eliminar(int id). Sin código JDBC.
Implementación
AlumnoDAOImpl.java — Lógica JDBC Implementa AlumnoDAO. Aquí está todo el código JDBC: PreparedStatement, ResultSet, gestión de excepciones, mapeo ORM.
Conexión
ConexionBD.java — Fábrica de conexiones Centraliza la URL del driver y el método getConnection(). Si se cambia de SQLite a MySQL, solo se modifica esta clase.
AlumnoDAO.java — Interfaz DAO completa 
import java.util.List;

public interface AlumnoDAO {
    boolean       insertar(Alumno alumno);
    List<Alumno>  listarTodos();
    Alumno        obtenerPorId(int id);
    boolean       actualizar(Alumno alumno);
    boolean       eliminar(int id);
}
Ventaja del patrón DAO: el resto de la aplicación solo conoce la interfaz AlumnoDAO. Puedes cambiar SQLite por MySQL, o crear una implementación de prueba en memoria, sin modificar ninguna otra clase.

Actividad evaluable — CRUD completo con patrón DAO

Implementa una aplicación Java de consola para gestionar un catálogo de productos de una tienda. La clase Producto tendrá los atributos: id, nombre, categoria, precio y stock.

1
Crea la clase Producto (POJO) con todos sus atributos, constructor vacío, constructor con parámetros, getters y setters.
2
Implementa ConexionBD con la URL de SQLite y el método estático getConnection(). Crea la tabla productos al arrancar si no existe.
3
Define la interfaz ProductoDAO con los métodos: insertar, listarTodos, obtenerPorId, actualizar, eliminar y buscarPorCategoria.
4
Implementa ProductoDAOImpl. Usa PreparedStatement en todos los métodos y try-with-resources para gestionar los recursos.
5
Crea una clase Menu con un menú interactivo de consola: listar, insertar, actualizar precio/stock, eliminar por id, buscar por categoría y salir.
6
(Ampliación) Añade una segunda implementación ProductoDAOMemoria que almacene los datos en una lista en memoria para poder ejecutar pruebas unitarias sin necesitar la base de datos.
Criterios de evaluación: uso correcto de PreparedStatement, cierre de recursos con try-with-resources, separación correcta de capas (Modelo / DAO / Menú), manejo de excepciones, y capturas de pantalla de todas las operaciones en funcionamiento.
