Boletín de Ejercicios – Ficheros de Texto en Java

Objetivo: practicar lectura y escritura de ficheros de texto, FileReader / FileWriter, BufferedReader / BufferedWriter, procesamiento secuencial, búsqueda, conteo, comparación de archivos y persistencia sencilla. Recomendación: crea un paquete por actividad o un proyecto con subpaquetes.

Actividad 1 📄 Mostrar el contenido de un fichero lectura básica

Realiza un programa que solicite al usuario el nombre de un fichero de texto y muestre su contenido en pantalla.

  • Si el usuario no introduce ningún nombre, usa por defecto prueba.txt.
  • Muestra el contenido línea a línea.
Actividad 2 👤 Guardar nombre y edad escritura básica

Diseña una aplicación que pida al usuario su nombre y su edad.

  • Guarda estos datos en el fichero datos.txt.
  • Si el fichero ya existe, debe borrarse su contenido antes de escribir.
  • Si no existe, debe crearse.
Actividad 3 📑 Duplicar ficheros de texto copias

Crea un programa que duplique el contenido de un fichero. Realiza dos versiones:

  • a) Duplicar original.txt en copia.txt.
  • b) Pedir el nombre del fichero fuente y copiarlo en otro con el mismo nombre, añadiendo el prefijo copia_de_.
Actividad 4 🔢 Contar caracteres, líneas y palabras procesamiento

Realiza un programa que lea un fichero de texto llamado carta.txt y cuente:

  • número de caracteres
  • número de líneas
  • número de palabras

Para facilitar el procesamiento, supondremos que cada palabra está separada de otra por un único espacio en blanco.

Actividad 5 📈 Menor y mayor valor en un fichero números

En el archivo numeros.txt disponemos de una serie de números, uno por cada línea.

Diseña un programa que procese el fichero y muestre:

  • el menor número
  • el mayor número
Actividad 6 ✍️ Libro de firmas persistencia

Crea una aplicación para gestionar un libro de firmas almacenado en firmas.txt.

  • Debe permitir mostrar todas las firmas.
  • Debe permitir insertar un nuevo nombre.
  • Antes de insertar, comprueba que el nombre no esté repetido.
Actividad 7 📚 Comando “more” lectura paginada

En Linux existe el comando more, que muestra un fichero poco a poco.

Implementa un programa que funcione de forma similar:

  • muestra el contenido de un fichero de texto
  • cada 24 líneas debe pausar la salida
  • el usuario decidirá si quiere continuar
Actividad 8 👥 Mezclar dos listados ordenados fusión ordenada

Disponemos de dos ficheros, perso1.txt y perso2.txt, ambos con nombres de personas ordenados alfabéticamente.

Realiza un programa que:

  • lea ambos ficheros
  • genere un tercer fichero llamado todos.txt
  • guarde en él todos los nombres ordenados alfabéticamente
Actividad 9 🔐 Encriptador con alfabeto de sustitución codificación

Diseña un programa que lea un fichero de texto, lo codifique y cree un nuevo archivo con el mensaje cifrado.

El alfabeto de codificación se encontrará en el fichero codec.txt.

Ejemplo de alfabeto:

Alfabeto: a b c d e f g h i j k l m n o p q r s t u v w x y z
Cifrado : e m s r c y j n f x i w t a k o z d l q v b h u p g
Actividad 10 🔓 Decodificador decodificación

Utilizando el fichero codec.txt del ejercicio anterior, diseña un programa decodificador.

  • Debe leer un fichero codificado.
  • Debe reconstruir el texto original.
Actividad 11 🔲 Transpuesta de una matriz matrices

El fichero matriz.txt contiene los datos de una matriz cuadrada.

Realiza un programa que:

  • lea la matriz desde el fichero
  • muestre por pantalla su matriz transpuesta
Actividad 12 ⚖️ Comparar dos archivos comparación

Algunos sistemas operativos disponen del comando comp, que compara dos archivos e indica si son iguales o distintos.

Diseña este comando de forma que, además:

  • indique en qué línea aparece la primera diferencia
  • indique en qué carácter aparece esa diferencia

Utiliza los ficheros texto1.txt y texto2.txt.

Actividad 13 ☎️ Agenda persistente proyecto mini

Diseña una pequeña agenda con el siguiente funcionamiento:

1. Nuevo contacto
2. Buscar por nombre
3. Mostrar todos
4. Salir

Condiciones de la agenda:

  • Guardará nombre y teléfono de un máximo de 20 personas.
  • La opción 1 permite insertar un nuevo contacto siempre que la agenda no esté llena.
  • No se deben permitir nombres repetidos.
  • La opción 2 debe mostrar todos los teléfonos cuyos nombres coincidan con una cadena parcial. Ejemplo: si se busca Pe, puede mostrar Pedro, Pepe y Petunia.
  • La opción 3 mostrará todos los contactos ordenados alfabéticamente por el nombre.
  • La opción 4 guardará todos los datos en el fichero agenda.txt.
  • La próxima vez que se ejecute el programa, debe cargar automáticamente los datos guardados.
Actividad 14 🧮 Contador de palabras mejorado tokenización

Repite el ejercicio de contar palabras, pero ahora sabiendo que una palabra no está separada de otra únicamente por un espacio en blanco.

Considera como separadores:

  • espacio en blanco
  • tabuladores
  • punto
  • coma
  • punto y coma
  • paréntesis
Actividad 15 🔎 Buscar ocurrencias en un fichero búsqueda textual

Diseña un programa al que se le proporcione:

  • el nombre de un fichero de texto
  • una cadena de búsqueda

Debe localizar todas las ocurrencias de la cadena en el fichero, indicando:

  • la fila donde aparece
  • la columna donde aparece
Actividad 16 🧹 Eliminar comentarios de código Java procesamiento de código

Escribe un programa que pida el nombre de un fichero de texto que contenga código fuente en Java.

El programa debe crear un nuevo fichero cuyo nombre sea el original con el prefijo sin_comentarios_.

El contenido del nuevo fichero será el código fuente original, pero sin ningún tipo de comentario.

Actividad 17 ✂️ Particionar un fichero grande fragmentación

Crea una aplicación a la que se le proporciona:

  • un fichero de texto
  • un tamaño máximo para cada fragmento

El tamaño podrá expresarse en:

  • b → bytes
  • k → kilobytes
  • m → megabytes

La aplicación debe fragmentar el fichero original en tantos ficheros como sea necesario. Los nombres de los nuevos ficheros tendrán el prefijo parte999_, donde 999 será el número de volumen.

Actividad 18 🧩 Reconstruir un fichero fragmentado recomposición

Escribe un programa que, a partir de los ficheros generados en el ejercicio anterior, reconstruya el fichero original.

  • Debe leer todas las partes en el orden correcto.
  • Debe generar nuevamente el fichero completo.
