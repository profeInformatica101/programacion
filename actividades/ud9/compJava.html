// Interfaces de comparación

Comparable vs Comparator

Ordena objetos en Java con criterio propio — entiende cuándo usar el orden natural de una clase y cuándo externalizar la lógica.

Java 8+ java.lang.Comparable java.util.Comparator Arrays.sort()
01 — Concepto

¿Qué diferencia a cada uno?

Comparable
  • Define el orden natural del objeto
  • Se implementa dentro de la clase
  • Método: compareTo(T o)
  • La clase "sabe" cómo ordenarse a sí misma
Comparator
  • Define un orden alternativo
  • Se implementa fuera de la clase
  • Método: compare(T a, T b)
  • Permite múltiples criterios sin tocar la clase
02 — Comparable

Implementando Comparable

Java — Producto.java 
public class Producto implements Comparable<Producto> {

    private String  nombre;
    private double  precio;
    private int     stock;

    // Constructor, getters… (omitidos por brevedad)

    @Override
    public int compareTo(Producto otro) {
        // 1.º criterio: precio ascendente
        int cmp = Double.compare(this.precio, otro.precio);
        if (cmp != 0) return cmp;

        // Desempate: nombre alfabético
        return this.nombre.compareTo(otro.nombre);
    }
}

Regla del signo: compareTo devuelve negativo si this va antes, cero si son iguales, positivo si va después. Usar Double.compare() e Integer.compare() evita los errores de desbordamiento de la resta manual.

03 — Comparator

Implementando Comparator

Java — lambda / clase anónima 
// ── Opción A: lambda (Java 8+) ──────────────────────────────
Comparator<Producto> porStock =
    (a, b) -> Integer.compare(a.getStock(), b.getStock());

// ── Opción B: Comparator.comparing (más expresivo) ──────────
Comparator<Producto> porNombre =
    Comparator.comparing(Producto::getNombre);

// ── Encadenado: precio desc → nombre asc ────────────────────
Comparator<Producto> mixto =
    Comparator.comparingDouble(Producto::getPrecio)
               .reversed()
               .thenComparing(Producto::getNombre);

// ── Uso con Arrays.sort ─────────────────────────────────────
Producto[] catalogo = /* … */;
Arrays.sort(catalogo, mixto);

05 — Ordenación manual

Bubble sort usando compareTo

Java — algoritmo con Comparable 
/**
 * Bubble sort genérico — T debe implementar Comparable.
 * OJO: nunca modifiques el array original en el examen;
 *      haz una copia con Arrays.copyOf() primero.
 */
public static <T extends Comparable<T>> void bubbleSort(T[] arr) {
    T[] copia = Arrays.copyOf(arr, arr.length);
    int n = copia.length;
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (copia[j].compareTo(copia[j + 1]) > 0) {
                T tmp     = copia[j];
                copia[j]     = copia[j + 1];
                copia[j + 1] = tmp;
            }
        }
    }
    // Aquí usarías 'copia' para el resultado
}

⚠ Errores típicos

  • 01 Invertir el signo en compareTo — el array queda al revés
  • 02 No contemplar desempates — orden inestable entre iguales
  • 03 Olvidar tratar null — NullPointerException en runtime
  • 04 Modificar el array original en vez de trabajar con una copia
  • 05 Usar a - b en lugar de Integer.compare(a,b) — overflow

Regla de decisión rápida

✓ Orden único → Comparable
✓ Varios criterios → Comparator
✓ Clase ajena → Comparator
