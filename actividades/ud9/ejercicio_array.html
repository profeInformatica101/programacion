Boletín de Ejercicios – Interfaces (POO) en Java Versión con Arrays

Objetivo: practicar interfaces, polimorfismo, default/static methods, comparación y composición sin usar colecciones.

Regla: no se permite List, ArrayList, Collections, Comparator ni métodos como list.sort(...). Todo se hace con arrays y bucles.
Actividad 1 ✅ “Imprimible” (calentamiento) interfaces + arrays

Crea una interfaz Imprimible con un método:

String imprimir();

Implementa Imprimible en dos clases:

  • Factura (id, cliente, importe)
  • Informe (titulo, autor, paginas)

Crea un main que guarde objetos en un Imprimible[] e imprima imprimir() de todos.

for (int i = 0; i < docs.length; i++) {
  System.out.println(docs[i].imprimir());
}
Actividad 2 🎮 “Daño” en un mini-juego polimorfismo + arrays

Define una interfaz Danable:

void recibirDanio(int puntos);

Crea dos clases:

  • Jugador (vida, nombre)
  • Enemigo (vida, tipo)

Haz que ambas implementen la interfaz y que reduzcan su vida.

En main, guarda varios Danable en un Danable[] y aplica daño en bucle.

Extra: si la vida llega a 0, muestra un mensaje tipo “KO”.

Interfaz Conectable:

Prueba en main un Conectable[] conectando/desconectando y mostrando estado.

Actividad 5 🧾 “Descuentable” con métodos default default + arrays

Interfaz Descuentable:

double precioBase();
default double precioFinal(double porcentaje) {
  return precioBase() * (1 - porcentaje / 100.0);
}

Crea Producto (nombre, precio) y Servicio (descripcion, tarifa) implementando precioBase().

En main, muestra precios con y sin descuento usando polimorfismo con Descuentable[].

Actividad 6 📦 Ordenación de productos (con interfaz) interfaces + ordenación manual

1) Crea la interfaz Valorable:

double getPrecio();
String getNombre();

2) Crea la clase Producto con atributos codigo, nombre y precio.

3) Obligatorio: la clase Producto debe implementar la interfaz Valorable (es decir, debe implementar los métodos getPrecio() y getNombre()).

4) En main: crea un array de productos (Producto[]), trátalo como Valorable[] y ordénalo por:

  • precio ascendente
  • extra: si el precio empata, desempata por nombre (A–Z)

La ordenación debe hacerse con bucles (burbuja o selección) y sin colecciones.

// Idea (burbuja):
for (int i = 0; i < productos.length - 1; i++) {
  for (int j = 0; j < productos.length - 1 - i; j++) {
    // si productos[j] debe ir después de productos[j+1], intercambiar
  }
}

Extra (opcional): crea un método estático que reciba un Valorable[]:

static void ordenarPorPrecioAscYNombre(Valorable[] arr) { ... }
Actividad 7 🏁 Ordenación con Comparable, Comparator y una interfaz de apoyo Comparable + Comparator + interfaces + arrays

Clase a reutilizar: usa la clase Producto.

Interfaz relacionada con la práctica: crea una interfaz Mostrable con el método:

String mostrar();

Haz que la clase Producto implemente:

  • Comparable<Producto>, para definir su orden natural.
  • Mostrable, para poder mostrar su información de forma uniforme.

Apartado a) Comparable: define el orden natural de Producto por nombre A–Z. Si dos productos tienen el mismo nombre, desempata por precio ascendente.

Apartado b) Comparator: crea un Comparator<Producto> para ordenar por precio descendente.

Apartado c) Comparator: crea otro Comparator<Producto> para ordenar por código A–Z.

Apartado d) Array y pruebas: trabaja con un Producto[] y realiza lo siguiente en main:

  • Muestra el array original usando el método mostrar().
  • Ordénalo con Arrays.sort(productos) usando el Comparable.
  • Ordénalo con Arrays.sort(productos, comparadorPrecioDesc).
  • Ordénalo con Arrays.sort(productos, comparadorCodigoAZ).
  • Muestra el resultado tras cada ordenación.

Extra: permite elegir el criterio desde un menú por consola:

  • 1: orden natural por nombre A–Z
  • 2: precio descendente
  • 3: código A–Z
  • 4: mostrar todos
Regla importante: en esta actividad deben usarse obligatoriamente Comparable, Comparator y la interfaz Mostrable. La ordenación se realizará con Arrays.sort(...).
Actividad 8 🧠 “Validable” y “Mostrable” (reglas de negocio) interfaces + excepciones + polimorfismo

Define una interfaz Validable con el método:

void validar() throws IllegalArgumentException;

Reutiliza también la interfaz Mostrable de la actividad anterior:

String mostrar();

Crea las clases:

  • Usuario (email, password)
  • Reserva (fecha, numPersonas)

Haz que ambas clases implementen Validable y Mostrable.

Implementa validar() aplicando reglas de negocio, por ejemplo:

  • el email debe contener @,
  • la contraseña debe tener al menos 8 caracteres,
  • el número de personas debe ser mayor que 0.

En main, guarda los objetos en un Validable[], valida cada uno capturando excepciones y muestra mensajes claros. Después, muestra su información llamando a mostrar().

Actividad 9 🎵 “Reproducible” y “Mostrable” (interfaces y composición) composición + polimorfismo

Define una interfaz Reproducible con los métodos:

void play();
void pause();
void stop();

Reutiliza también la interfaz Mostrable.

Crea las clases Cancion y Podcast, ambas implementando Reproducible y Mostrable.

Crea una clase Reproductor que reciba un Reproducible y lo controle con los métodos play(), pause() y stop().

En main:

  • Crea varias pistas.
  • Muestra su información con mostrar().
  • Reprodúcelas usando un objeto Reproductor.

Opcional: guarda varias pistas en un Reproducible[] y cambia el elemento actual por índice.

Actividad 11 🗂️ Repositorio genérico (contrato) genéricos + arrays + interfaces

Define una interfaz genérica Repositorio<T>:

void add(T item);
boolean remove(T item);
T findById(String id);
int size();

Para poder buscar por identificador, define la interfaz:

interface Identificable {
  String getId();
}

Además, para mostrar los elementos recuperados, reutiliza la interfaz Mostrable.

Implementa RepositorioMemoria<T extends Identificable> usando un array interno y un contador.

  • Empieza con un array de tamaño fijo, por ejemplo 10.
  • Opcionalmente, si se llena, crea uno nuevo más grande y copia los elementos.

Prueba el repositorio con una clase Alumno o Producto que implemente Identificable y Mostrable.

Actividad 13 🧱 “Copiable” (copia profunda con arrays) diseño + arrays + interfaces

Diseña una interfaz genérica Copiable<T>:

T copia();

Reutiliza la interfaz Mostrable.

Crea una clase Documento con:

  • titulo
  • contenido
  • etiquetas de tipo String[]

Haz que implemente Copiable<Documento> y Mostrable.

La copia debe ser profunda para el array de etiquetas, es decir, debe crearse un array nuevo y copiarse elemento a elemento.

En main, crea un documento, haz una copia, modifica las etiquetas del original y comprueba que la copia no cambia.
