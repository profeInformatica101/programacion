Boletín de Ejercicios – Interfaces (POO) en Java Versión con Arrays

Objetivo: practicar interfaces, polimorfismo, default/static methods, comparación y composición sin usar colecciones.

Regla: no se permite List, ArrayList, Collections, Comparator ni métodos como list.sort(...). Todo se hace con arrays y bucles.
Actividad 1 ✅ “Imprimible” (calentamiento) interfaces + arrays

Crea una interfaz Imprimible con un método:

String imprimir();

Implementa Imprimible en dos clases:

  • Factura (id, cliente, importe)
  • Informe (titulo, autor, paginas)

Crea un main que guarde objetos en un Imprimible[] e imprima imprimir() de todos.

for (int i = 0; i < docs.length; i++) {
  System.out.println(docs[i].imprimir());
}
Actividad 2 🎮 “Daño” en un mini-juego polimorfismo + arrays

Define una interfaz Danable:

void recibirDanio(int puntos);

Crea dos clases:

  • Jugador (vida, nombre)
  • Enemigo (vida, tipo)

Haz que ambas implementen la interfaz y que reduzcan su vida.

En main, guarda varios Danable en un Danable[] y aplica daño en bucle.

Extra: si la vida llega a 0, muestra un mensaje tipo “KO”.

Actividad 3 🔌 “Conectable” (simulación de dispositivos) estado + arrays

Interfaz Conectable:

void conectar();
void desconectar();
boolean estaConectado();

Implementa en Wifi, Bluetooth y Cable con un atributo de estado.

Prueba en main un Conectable[] conectando/desconectando y mostrando estado.

Actividad 4 💳 Pagos (estrategia) Strategy

Interfaz MetodoPago:

boolean pagar(double importe);

Implementa:

  • Tarjeta (saldo disponible, limite)
  • Bizum (saldo, telefono)
  • Efectivo (dinero en cartera)

Crea una clase Caja con un método cobrar(MetodoPago mp, double importe) que delegue el pago y registre el resultado.

Nota: esta actividad no necesita arrays, pero puedes probar varios pagos con un array de métodos de pago si quieres.

Actividad 5 🧾 “Descuentable” con métodos default default + arrays

Interfaz Descuentable:

double precioBase();
default double precioFinal(double porcentaje) {
  return precioBase() * (1 - porcentaje / 100.0);
}

Crea Producto (nombre, precio) y Servicio (descripcion, tarifa) implementando precioBase().

En main, muestra precios con y sin descuento usando polimorfismo con Descuentable[].

Actividad 6 📦 Ordenación de productos (con interfaz) interfaces + ordenación manual

1) Crea la interfaz Valorable:

double getPrecio();
String getNombre();

2) Crea la clase Producto con atributos codigo, nombre y precio.

3) Obligatorio: la clase Producto debe implementar la interfaz Valorable (es decir, debe implementar los métodos getPrecio() y getNombre()).

4) En main: crea un array de productos (Producto[]), trátalo como Valorable[] y ordénalo por:

  • precio ascendente
  • extra: si el precio empata, desempata por nombre (A–Z)

La ordenación debe hacerse con bucles (burbuja o selección) y sin colecciones.

// Idea (burbuja):
for (int i = 0; i < productos.length - 1; i++) {
  for (int j = 0; j < productos.length - 1 - i; j++) {
    // si productos[j] debe ir después de productos[j+1], intercambiar
  }
}

Extra (opcional): crea un método estático que reciba un Valorable[]:

static void ordenarPorPrecioAscYNombre(Valorable[] arr) { ... }
Actividad 7 🏁 Ordenación alternativa (con interfaz) polimorfismo + arrays

Interfaz a usar: reutiliza la interfaz Valorable de la actividad 6.

Obligatorio: vuelve a usar un array de Producto, pero trabajando mediante el tipo Valorable[] (polimorfismo).

Crea dos métodos estáticos de ordenación que reciban un Valorable[]:

  • ordenarPorNombreAZ(Valorable[] arr) (A–Z)
  • ordenarPorPrecioDesc(Valorable[] arr) (mayor → menor)

En main:

  • Muestra el array original
  • Ordena por nombre A–Z y muestra
  • Ordena por precio descendente y muestra

Extra: permite elegir el orden desde un menú por consola (1/2/3):

  • 1: Nombre A–Z
  • 2: Precio DESC
  • 3: Ambos (mostrar ambos resultados)
Regla: sigue sin permitirse Comparator ni Collections. La comparación se hace “a mano” usando getPrecio() y getNombre() definidos en Valorable.
Actividad 8 🧠 “Validable” (reglas de negocio) interfaces + excepciones + arrays

Interfaz Validable:

void validar() throws IllegalArgumentException;

Crea:

  • Usuario (email, password)
  • Reserva (fecha, numPersonas)

Implementa validar() con reglas (ej.: email contiene “@”; password mínimo 8; numPersonas > 0).

En main valida un Validable[] capturando excepciones y mostrando mensajes claros.

Actividad 9 🎵 “Reproducible” (interfaces y composición) composición

Interfaz Reproducible:

void play();
void pause();
void stop();

Crea clases Cancion y Podcast que implementen la interfaz.

Crea una clase Reproductor que reciba un Reproducible y lo controle (play/pause/stop).

En main, cambia de pista varias veces.

Opcional: guarda varias pistas en un Reproducible[] e intercambia el elemento actual por índice.

Actividad 10 🧩 Interfaces múltiples (roles) multi-implements

Define:

  • interface Trabajador { void trabajar(); }
  • interface Estudiante { void estudiar(); }

Crea clases:

  • Persona (nombre) – clase base
  • Becario que extienda Persona e implemente Trabajador y Estudiante

En main, trata el objeto como Trabajador y como Estudiante (referencias distintas) y llama a los métodos.

Actividad 11 🗂️ Repositorio genérico (contrato) genéricos + arrays

Interfaz genérica Repositorio<T>:

void add(T item);
boolean remove(T item);
T findById(String id);
int size();

Para poder hacer findById, define una interfaz:

interface Identificable { String getId(); }

Implementa RepositorioMemoria<T extends Identificable> usando un array interno y un contador.

  • Empieza con un array de tamaño fijo (ej.: 10).
  • Cuando se llene, puedes (opcional) crear otro array más grande y copiar.

Prueba con una clase Alumno o Producto.

Actividad 12 🧪 “Loggeable” con método static static methods

Crea interfaz Loggeable con:

void log(String msg);

static String now() {
  return java.time.LocalDateTime.now().toString();
}

Implementa log en ConsolaLogger (imprime por consola con fecha).

En main, registra eventos de varias clases usando Loggeable.

Extra: añade niveles INFO/WARN/ERROR (como texto) y formatea el mensaje.

Actividad 13 🧱 “Copiable” (copia profunda con arrays) diseño

Diseña una interfaz Copiable<T>:

T copia();

Crea una clase Documento (titulo, contenido, etiquetas: String[]) que implemente Copiable<Documento>.

La copia debe ser “profunda” para el array de etiquetas (crear un array nuevo y copiar elemento a elemento).

En main, crea un documento, cópialo, modifica etiquetas en el original y comprueba que la copia no cambia.

Actividad 14 🧠 Proyecto mini: “Motor de reglas” (sin colecciones) nivel pro + arrays

Crea una interfaz Regla<T>:

boolean cumple(T obj);
String mensajeError();

Crea una clase Validador<T> que contenga un Regla<T>[] y tenga:

String[] validar(T obj);

Implementa varias reglas para Usuario (email válido, password fuerte, edad mínima, etc.).

Si falla, devuelve un array de errores (puedes devolver un array con tamaño fijo y un contador, o devolver un array ajustado al final).

Extra: crea reglas con clases anónimas si te apetece.
