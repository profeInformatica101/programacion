Boletín de Ejercicios – Interfaces (POO) en Java

Objetivo: practicar interfaces, polimorfismo, default/static methods, comparadores y composición. Recomendación: crea un paquete por actividad o un proyecto con subpaquetes.

Actividad 1 ✅ “Imprimible” (calentamiento) interfaces básicas

Crea una interfaz Imprimible con un método:

String imprimir();

Implementa Imprimible en dos clases:

  • Factura (id, cliente, importe)
  • Informe (titulo, autor, paginas)

Crea un main que guarde objetos en un List<Imprimible> e imprima imprimir() de todos.

Actividad 2 🎮 “Daño” en un mini-juego polimorfismo

Define una interfaz Danable:

void recibirDanio(int puntos);

Crea dos clases:

  • Jugador (vida, nombre)
  • Enemigo (vida, tipo)

Haz que ambas implementen la interfaz y que reduzcan su vida. En main, guarda varios Danable y aplica daño en bucle.

Extra: si la vida llega a 0, muestra un mensaje tipo “KO”.

Actividad 3 🔌 “Conectable” (simulación de dispositivos) estado

Interfaz Conectable:

void conectar();
void desconectar();
boolean estaConectado();

Implementa en Wifi, Bluetooth y Cable con un atributo de estado. Prueba en main una lista de Conectable conectando/desconectando y mostrando estado.

Actividad 4 💳 Pagos (estrategia) Strategy

Interfaz MetodoPago:

boolean pagar(double importe);

Implementa:

  • Tarjeta (saldo disponible, limite)
  • Bizum (saldo, telefono)
  • Efectivo (dinero en cartera)

Crea una clase Caja con un método cobrar(MetodoPago mp, double importe) que delegue el pago y registre el resultado.

Actividad 5 🧾 “Descuentable” con métodos default default methods

Interfaz Descuentable:

double precioBase();
default double precioFinal(double porcentaje) {
  return precioBase() * (1 - porcentaje / 100.0);
}

Crea Producto (nombre, precio) y Servicio (descripcion, tarifa) implementando precioBase().

En main, muestra precios con y sin descuento usando polimorfismo (List<Descuentable>).

Actividad 6 📦 Comparación de productos Comparable

Crea una clase Producto (codigo, nombre, precio). Implementa Comparable<Producto> para ordenar por precio ascendente.

En main crea una lista de productos, ordénala con Collections.sort y muéstrala.

Extra: si el precio empata, desempata por nombre.

Actividad 7 🏁 Ordenación de productos Comparable + Comparator

Con la clase Producto de la actividad anterior, realiza los siguientes apartados:

  • a) Comparable: haz que Producto implemente Comparable<Producto> para definir su orden natural por nombre (A-Z).
  • b) Comparator: crea un Comparator<Producto> para ordenar por precio descendente.
  • c) Comparator: crea otro Comparator<Producto> para ordenar por código (A-Z).
  • d) Pruebas en main: crea una lista con varios productos, muéstrala sin ordenar, ordénala con Collections.sort(lista) usando el Comparable, y después vuelve a ordenarla con cada uno de los Comparator.

Extra: permite al usuario elegir por consola qué criterio aplicar: 1 = orden natural, 2 = precio descendente, 3 = código.

Actividad 8 🧠 “Validable” (reglas de negocio) interfaces + excepciones

Interfaz Validable:

void validar() throws IllegalArgumentException;

Crea:

  • Usuario (email, password)
  • Reserva (fecha, numPersonas)

Implementa validar() con reglas (ej.: email contiene “@”; password mínimo 8; numPersonas > 0). En main valida una lista de Validable capturando excepciones y mostrando mensajes claros.

Actividad 9 🎵 “Reproducible” (interfaces y composición) composición

Interfaz Reproducible:

void play();
void pause();
void stop();

Crea clases Cancion y Podcast que implementen la interfaz. Crea una clase Reproductor que reciba un Reproducible y lo controle (play/pause/stop). Cambia de pista varias veces en main.

Actividad 10 🧩 Interfaces múltiples (roles) multi-implements

Define:

  • interface Trabajador { void trabajar(); }
  • interface Estudiante { void estudiar(); }

Crea clases:

  • Persona (nombre) – clase base
  • Becario que extienda Persona e implemente Trabajador y Estudiante

En main, trata el objeto como Trabajador y como Estudiante (referencias distintas) y llama a los métodos.

Actividad 11 🗂️ Repositorio genérico (contrato) genéricos

Interfaz genérica Repositorio<T>:

void add(T item);
boolean remove(T item);
T findById(String id);
int size();

Implementa RepositorioMemoria<T> usando una colección. Para poder hacer findById, define una interfaz:

interface Identificable { String getId(); }

y restringe el genérico: RepositorioMemoria<T extends Identificable>. Prueba con una clase Alumno o Producto.

Actividad 12 🧪 “Loggeable” con método static static methods

Crea interfaz Loggeable con:

void log(String msg);

static String now() {
  return java.time.LocalDateTime.now().toString();
}

Implementa log en ConsolaLogger (imprime por consola con fecha). En main, registra eventos de varias clases usando Loggeable.

Extra: añade niveles INFO/WARN/ERROR (como texto) y formatea el mensaje.

Actividad 13 🧱 “ClonableSeguro” (copia sin liarla) diseño

Diseña una interfaz Copiable<T>:

T copia();

Crea una clase Documento (titulo, contenido, etiquetas: List<String>) que implemente Copiable<Documento>. La copia debe ser “profunda” para la lista de etiquetas.

En main, crea un documento, cópialo, modifica etiquetas en el original y comprueba que la copia no cambia.

Actividad 14 🧠 Proyecto mini: “Motor de reglas” nivel pro

Crea una interfaz Regla<T>:

boolean cumple(T obj);
String mensajeError();

Crea una clase Validador<T> que contenga una lista de reglas y tenga:

List<String> validar(T obj);

Implementa varias reglas para Usuario (email válido, password fuerte, edad mínima, etc.). Si falla, devuelve lista de errores.

Extra: usa expresiones lambda para crear reglas rápidamente (si te apetece).
