Cuentas y bancos

Diseñar la clase CuentaCorriente, sabiendo que los datos necesarios son:

  • saldo
  • límite de descubierto
  • nombre y DNI del titular

Operaciones:

  • Crear la cuenta: se necesita nombre y DNI del titular. El saldo inicial será 0 y el límite de descubierto será de −50€.
  • Sacar dinero: solo se podrá sacar hasta el límite de descubierto. El método debe indicar si la operación ha sido posible.
  • Ingresar dinero: incrementa el saldo.
  • Mostrar información: muestra todos los datos de la cuenta corriente.

En la clase CuentaCorriente, sobrecargar los constructores para permitir:

  • Crear objetos solo con el saldo inicial; no serán necesarios los datos del titular. El límite de descubierto será por defecto 0€.
  • Crear objetos con saldo inicial, límite de descubierto y DNI del titular.

Escribir un programa que compruebe el funcionamiento de todos los métodos de CuentaCorriente, incluidos los constructores sobrecargados.

Modificar la visibilidad de la clase CuentaCorriente para que sea visible desde clases externas y la visibilidad de sus atributos para que:

  • saldo y limite no sean visibles desde otras clases.
  • nombre sea público para cualquier clase.
  • dni solo sea visible por clases del mismo paquete (clases vecinas).

Texto, dispositivos y hogar

Diseñar la clase Texto que gestione una cadena de caracteres:

  • La longitud máxima se especifica en el constructor.
  • Permite añadir un carácter al principio o al final si hay espacio disponible.
  • Permite añadir una cadena al principio o al final sin rebasar el tamaño máximo.
  • Debe poder indicar cuántas vocales (mayúsculas y minúsculas) contiene el texto.

Definir una clase que controle un sintonizador digital FM con:

  • Métodos up y down que suben/bajan 0.5 MHz.
  • Método display que muestra la frecuencia actual.
  • Rango de frecuencias entre 80 MHz y 108 MHz.
  • Frecuencia inicial: 80 MHz.
  • Si se sobrepasa un límite, se pasa al extremo contrario.

Escribir un programa principal para probarlo.

Modelar una casa con muchas bombillas. La clase Bombilla debe:

  • Tener una variable privada que indique si está encendida/apagada.
  • Tener un método que devuelva el estado de una bombilla.
  • Incorporar un interruptor general (fusible): si se apaga, todas las bombillas quedan apagadas.
  • Al restaurar el fusible, cada bombilla vuelve al estado que tenía antes.
  • Una bombilla responde que está encendida solo si:
    • su interruptor particular está activado y
    • el interruptor general permite el paso de corriente.
Modelos de dominio

Definir paquetes y clases (solo atributos y constructores) para gestionar una empresa ferroviaria con dos grandes grupos:

  • Personal: Maquinista, Mecanico, JefeEstacion (todas de uso público).
  • Maquinaria: Tren, Locomotora y Vagon.

Restricciones principales:

  • De cada tipo de personal se guardan los datos especificados (nombre, DNI, teléfono, rango, especialidad…).
  • Vagones: capacidad máxima, capacidad actual y tipo de mercancía.
  • Locomotoras: matrícula, potencia, antigüedad y mecánico asignado.
  • Trenes: una locomotora + hasta 5 vagones + maquinista responsable.
  • Desde clases externas solo se pueden construir objetos Tren y Locomotora. La clase Vagon será visible solo por sus clases vecinas.

Estructuras dinámicas de datos

Implementar la clase Lista de números Integer almacenados en una tabla redimensionable.

Métodos principales:

  • Constructor por defecto (capacidad 10) y otro con tamaño inicial.
  • Obtener número de elementos insertados.
  • Insertar al final, al principio o en un índice concreto.
  • Añadir al final los elementos de otra lista.
  • Eliminar elemento por índice.
  • Obtener elemento por índice.
  • Buscar número (devuelve índice o −1).
  • Representar la lista como cadena de caracteres.

Implementar la clase PilaTabla para números Integer usando una tabla interna.

Debe permitir al menos las operaciones fundamentales:

  • apilar()
  • desapilar()
  • consultar la cima
  • comprobar si está vacía

Repetir el ejercicio anterior implementando la pila con un objeto de la clase Lista (Ejercicio 12) en vez de con una tabla.

Implementar la clase ColaTabla para números Integer usando una tabla:

  • Los elementos se encolan al final y se desencolan al principio.
  • Gestionar índices de primero y último de la cola.

Repetir el ejercicio de ColaTabla, pero usando una Lista para guardar los elementos encolados.

Implementar la clase Conjunto usando una Lista de Integer:

  • Constructores (por defecto y con tamaño inicial).
  • Obtener número de elementos.
  • Insertar número (si ya existe, no se inserta).
  • Añadir los elementos de otro conjunto.
  • Eliminar un elemento concreto.
  • Eliminar todos los elementos que estén en otro conjunto.
  • Comprobar si un número pertenece al conjunto.
  • Representar el conjunto como cadena.

Añadir a Conjunto los métodos estáticos:

  • static boolean incluido(Conjunto c1, Conjunto c2)
  • static Conjunto union(Conjunto c1, Conjunto c2)
  • static Conjunto interseccion(Conjunto c1, Conjunto c2)

Aplicación de gestión

En un puesto fronterizo se recoge diariamente información del tránsito de personas (DNI y nombre completo).

Diseñar una aplicación con el siguiente menú:

  1. Paso de frontera.
  2. Mostrar todas las personas.
  3. Búsqueda por nombre.
  4. Búsqueda por DNI.
  5. Salir.

Cada opción realizará las acciones descritas, utilizando una o varias clases de apoyo para almacenar y gestionar los datos de las personas.
